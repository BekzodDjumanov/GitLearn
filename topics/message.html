Commit Messages

Commit messages are short descriptions that explain the changes made in a commit. They help developers understand the history and purpose of changes in a project. It's best practice to include descriptive messages so each commit is documented properly.

Importance of Commenting

Minimal Commit Messages

  • "bug"
  • "unit tests"
  • "README"

Adequate Commit Messages

  • "Fix bug in user login validation, lowering api verification expense (document expense decrease on company statement)"
  • "Add unit tests for payment module (add extra overhead costs to company statement)"
  • "Update README with setup instructions (roll out company tweet)"

Summary

Maintaining proper commit messages provide meaningful descriptions for rollouts or updates.
