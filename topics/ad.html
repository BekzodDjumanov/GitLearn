Archiving & Deleting Repositories

Archiving and deleting is the act of terminating specific repositories within a GitHub account. Archiving temporarily disables a repository (read only), while deleting indefinitely omits them from an account.

Archive

  • Purpose: Keep the code available without allowing further commits, issues, or pull requests.
  • When to Use: Project is no longer maintained but should remain accessible.
  • Key Points: Anyone can still fork or clone an archived repo.

Delete

  • Purpose: Remove outdated, private, or unused repositories entirely.
  • When to Use: Project is obsolete or contains sensitive information.
  • Key Points: once deleted, the repository is gone unless you have a local copy or backup.

Steps to Delete a Repository

1. Go to the repository’s Settings tab.

2. Scroll to the Danger Zone section.

3. Click Archive this repository.

4. Confirm the action.

Steps to Archive a Repository

1. Go to the repository’s Settings tab.

2. Scroll to Danger Zone.

3. Click Delete this repository.

4. Type the repository name to confirm.

Best Practices

Archive instead of delete if you might need the code later.

Always make a local backup before deleting.

Keep sensitive data (like API keys) out of repos before archiving or deleting.

Summary

Archiving and deleting repositories is when developers store away their code either temporarily or permanently, respectively.
