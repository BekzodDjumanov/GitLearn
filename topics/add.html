Add

The git add command updates the staging area (also called the index) by adding changes from your working directory. It does not save changes permanently—that’s what git commit does—but it marks specific files or changes for inclusion in the commit snapshot.

  • Staging Area: Moves changes from your working directory to Git’s staging area (index).
  • Selective Staging: You can choose specific files or even parts of files to stage.
  • Selective Staging: Can stage new files, track previously untracked files, and stage modifications or deletions.

Common Commands

Stage a single file

$ git add file.txt

Stage all changes in the current directory

$ git add .

Stage all JavaScript files

$ git add *.js

Interactively stage parts of files

$ git add -p

Summary

git add stages changes from your working directory, marking files or modifications to be included in the next commit. It helps you select what updates to save in your project’s history.
