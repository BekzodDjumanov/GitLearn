Remote Repositories

Remote repositories are versions of a project or development that are hosted on the internet or a network. This allows users to collaborate seamlessly.

Common Hosting Platforms

  • GitHub – Most popular hosting platform for open-source and private projects.
  • GitLab – Great for self-hosting and offers CI/CD components.
  • Bitbucket – Supports Git and Mercurial, ideal for large-scale environments.

Local vs Remote Repositories

Aspect

Local Repository

Remote Repository

Location

Stored on your personal computer or local machine

Stored on a server or cloud platform like GitHub, GitLab, or Bitbucket

Access

Accessible only on your machine unless shared manually

Accessible to anyone with the proper permissions over the internet

Collaboration

Limited to people who can access your machine directly

Allows multiple people to work together from anywhere

Backup

No backup unless you create it manually

Serves as a backup copy of your project online

Synchronization

Changes stay local until pushed to a remote

Updates require pulling from or pushing to local repositories

Common Commands

Connects local repo to remote

$ git remote add <name> <url>

Shows connections to local repo

$ git remote -v

Sends commits from local to remote branch

$ git push <remote> branch>

Fetches changes from remote and merges them into local branch

$ git pull <remote> branch>

Download changes from remote without merging

$ git fetch <remote>

Clones local copy of remote repo

$ git clone <url>

Summary

Remote repositories are hosted virtually, allowing others to efficiently collaborate with your visions.
