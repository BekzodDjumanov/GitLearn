Git Revert

Reverting is the process where a new commit undoes previous changes done by another (commit). "git revert" is a safe and encouraged command that conserves clear project history (superb for collaboration).

  • Safely undo changes without rewriting history.
  • Preserve a clear project history (good for collaboration).
  • Can selectively undo commits.

Important Feature

You can revert multiple commits either one by one or within a specified commit range.

Common Commands

Revert a single commit

$ git revert <commit-hash>

Revert multiple commits (from newest to oldest)

$ git revert <newest-commit>^..<oldest-commit>

Edit commit hash

$ git revert --edit <commit-hash>

Comparison

Purpose

Undo commits safely on shared branches

Summary

"git revert" is a command used to safely undo changes introduced by previous commits by creating new commits that reverse those changes.
