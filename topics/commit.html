Commit

"Comitting" is a command that allows users to save the current contents inside of a project. It records modifications to tracked files along with a descriptive message, creating a permanent checkpoint in the repositoryâ€™s history.

  • Snapshot of changes: Records the state of all tracked files at that moment.
  • Commit message: A short description explaining what changes were made and why.
  • Unique identifier (hash): Each commit is assigned a SHA-1 hash to distinguish it.

Example

$ git commit -m "commit message"

Summary

Committing inside of a project is an essential component in project building as it allows all changes to be saved and recorded.
