Issues & Discussions

Issues & discussions on GitHub act as a way for other developers to collaborate and assist each other. It is a built-in tracking system for bugs and questions, letting users comment things of substance.

Key Features of Issues

  • Title & Description: Explain the problem or idea clearly.
  • Labels: Categorize issues (e.g., bug, enhancement, documentation).
  • Assignees: Assign team members responsible for resolving it.
  • Milestones: Link issues to a broader project goal or release cycle.

Key Features of Discussions

  • Categories: Organize discussions into Ideas, Q&A, Show and Tell, etc.
  • Open-ended: No requirement for a resolution like an Issue.
  • Community Driven: Anyone with access can participate and share insights.

Best Practices

Respect

Use Issues for actionable tasks and trackable work.

Courtesy

Approach discussions and/or issues with an open-mind, and articulate your thoughts clearly.

Awareness

Always research issues before posting one to ensure one hasn't been opened already.

Examples

Issue: “Fix login bug where password field isn’t clearing.”

Discussion: “What features should we add to the next release?”

Summary

Issues & Discussions allow users to interact with each other, facilitating and contributing to their goals.
