Reset

"Resetting" is a command that allows users to undo changes by moving the current branchâ€™s HEAD to a specified state. It can unstage files, revert commits, or even discard changes depending on the options used. In essence, git reset is applicable for reverting past changes.

Three Modes

  • --soft: Moves HEAD but leaves changes staged.
  • --mixed (default): Moves HEAD and unstages changes, but keeps them in the working directory.
  • --hard: Moves HEAD and discards ALL changes in working directory and staging area.

Common Commands

Unstage file.txt

$ git reset HEAD {file}.txt

Undo last commit but keep changes staged

$ git reset --soft HEAD~1

Undo last commit, unstage changes but keep them locally

$ git reset --mixed HEAD~1

Undo last commit and discard changes completely

$ git reset --hard HEAD~1

Summary

"git reset" is useful for rectifying mistakes or cleaning up commits before pushing.
