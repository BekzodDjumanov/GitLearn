Forking Repositories

Forking refers to creating a new repository from someone else's, copying everything within the original. The repository is placed within the creators GitHub account.

Use Case

It’s commonly used in open-source development so you can freely experiment, make changes, and then contribute those changes back to the original project.

Why Fork?

  • Unlike cloning, forking allows developers to create new and separate repositories under their GitHub account. This provides full control and is not dependent on the original.
  • Cloning creates a local copy of a repository that still points to the original remote — best for collaboration within the same project.

Usage and Application

1. Contributing to open-source projects via Pull Requests.

2. Customizing a project without interfering with the main repository.

3. Experimenting with features in a safe environment.

Common Workflow

1. Fork

2. Clone repo into local environment

$ git clone https://github.com/your-username/forked-repo.git

3. Commit & push your changes

$ git push origin main

4. Commit & push your changes

Open a Pull Request on GitHub to suggest your changes to the original repository.

5. After forking, you can set the original repository as an upstream remote to keep your fork in sync

git remote add upstream https://github.com/original-owner/original-repo.git

git pull upstream main

Summary

Forking allows developers to contribute to or utilize repositories.
