Filter Branch

git filter-branch is an advanced Git command used to rewrite history by applying filters to all or selected commits in a branch.

  • Removing sensitive data (e.g., passwords, API keys) from the entire history.
  • Changing commit author information.
  • Restructuring or renaming files throughout the repositoryâ€™s history.

git filter-branch is considered dangerous because it rewrites commit history. It has largely been replaced by git filter-repo for safety and performance.

Common Commands

Basic syntax

$ git filter-branch [options] [revision-range]

Removing a file

$ git filter-branch --tree-filter 'rm -f secrets.txt' HEAD

Filter-Branch Options

Force operation (overwrites existing backup)

$ --force

Removes empty commits after filtering

$ --prune-empty

Filter branches and tags

$ -- --all

Filter by branch

$ -- --branches

Filter by tag

$ -- --tags

Original references backup location

$ --original refs/original/

Summary

"git filter-branch" allows users to rewrite git history by commit.
