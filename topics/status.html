Status

The git status command shows the current state of the working directory and the staging area. It helps you see which changes have been staged, which haven’t, and which files aren’t being tracked by Git yet; It allows a user to assess the current health of a working directory.

  • Shows untracked files (new files not yet staged).
  • Lists modified files that are staged for the next commit.
  • Useful for understanding what’s ready to be committed and what’s still in progress.

Common Commands

General usage

$ git status

Provides output in short format

$ git status -s

"Verbose" (more detailed explanation including textual changes)

$ git status -v

Summary

"git status" displays the current state of your working directory.
