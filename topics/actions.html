GitHub Actions

GitHub Actions is a CI/CD platform integrated directly within GitHub, allowing users to facilitate their automation workflows and software deployments. It allows users to create workflows that builds and tests every pull request to ones repository, or deploy merged pull requests to production.

1. Workflow Files

Workflows are defined in YAML files inside the .github/workflows/ directory of your repository. Each file represents a workflow that runs on specific triggers.

.github/workflows/

2. Triggers

  • push → Runs when code is pushed to a branch.
  • pull_request → Runs on pull requests.
  • schedule → Runs at specific times using cron syntax.
  • workflow_dispatch → Manual trigger from GitHub UI.

3. Jobs and Steps

A workflow is made up of jobs, and each job contains multiple steps. Steps can run shell commands, scripts, or pre-built actions.

$ name: CI
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install Dependencies
        run: npm install
      - name: Run Tests
        run: npm test

4. Benefits

  • Automates testing and deployment.
  • Integrates directly with GitHub repositories.
  • Customizable and reusable workflows via actions.
  • Supports multiple operating systems and environments.

Summary

GitHub Actions facilitates developers workflows by autonomously building and testing their products.
