Rebase

Rebase is a powerful tool that takes the commits from one branch and reapplies them onto another branch, creating a linear, cleaner history without merge commits. Allowing users to create isolated environments for new projects and/or collaborations.

How it Works

Working example

main: A --- B --- C feature: D --- E

Running:

$git checkout feature $git rebase main

D and E (commits) are rewritten on C

main: A --- B --- C feature: D' --- E'
  • Cleaner, linear history, making it easier to read and follow.
  • Avoids unnecessary merge commits.
  • Lets you update your feature branch with the latest from main before merging.

"git rebase" overrides commit history, therefore its best practice to avoid rebasing shared branches

Cancelling a rebase

$git rebase --abort

Summary

Rebasing allows developers to create clean, linear branches by replaying commits onto a new base, preserving the commit history in a streamlined way.
