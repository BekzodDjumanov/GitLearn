Pipelines

Pipelines in CI/CD are automated workflows that streamline the development process. It orchestrates a series of steps to build, test, and deploy applications, ensuring consistent and frequent releases.

Key Components

  • Build – Compile code and create artifacts.
  • Test – Run automated tests to validate functionality.
  • Deploy – Push changes to staging or production environments.
  • Validate – Check for errors or merge conflicts before final deployment.
  • Feedback – Report results back to developers for action.

The Goal

To ensure new code is published and reaches its designated destination without manual intervention.

Common CI/CD Pipeline

# Developer pushes code to feature branch
$ git push origin feature/add-login

# GitHub Actions CI/CD pipeline triggers automatically
- Install dependencies
- Run automated tests
- Build the project
- Deploy to staging environment
- Notify developer of success/failure

# Once approved, merge to main triggers deployment to production

Summary

In essence, a pipeline is a set of automated tasks that transport code development to production.
