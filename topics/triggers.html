Triggers

CI/CD triggers indicate when pipelines are initiated, typically when set by specific instructions on how and when to run. Common triggers are code pushes or pull requests.

Common Triggers

  • Code Commits: A new commit pushed to a branch in the repository.
  • Pull Requests: Actions related to pull requests, such as creation, opening, closing, merging, or commenting.
  • Tags: Creation or deletion of Git tags.
  • Branches: Creation or deletion of branches.

Scheduled Triggers

Scheduled triggers can be set at specific time intervals (e.g daily, weekly, bimonthly).

Webhooks

Artifact Events: When a new artifact is created or updated in an artifact repository.

External API Calls: Receiving notifications from other systems or services.

Manual Triggers

Manual triggers are explicitly set by developers to trigger a pipeline or workflow when needed.

The Goal

To identify intentional or unintentional changes in behavior within a pipeline.

Why Triggers?

  • Run pipelines without manual intervention.
  • Automatically test or build code after changes.
  • Immediate feedback on commits or pull requests.
  • Keeps the main branch stable and deployable.

Summary

Triggers automatically start workflows or pipelines in response to specific events, ensuring timely and consistent execution.
