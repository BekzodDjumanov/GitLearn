Branches

Branches allow deviation in project configuration, enabling users to work within multiple outlets without affecting the main components / database. "git branch" is used to list, create, or delete branches in a Git repository.

  • Branches are lightweight pointers to commits.
  • The main (or master) branch is the default starting branch.
  • You must checkout a branch before working on it.

Common Commands

List branches

$ git branch

Create a new branch

$ git branch feature-name

Delete a branch

$ git branch -d branch-name

Rename a branch

$ git branch -m old-name new-name

Summary

A separate line of development in Git, allowing you to work on features or fixes without affecting the main codebase.
