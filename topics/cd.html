Continuous Deployment

Continuous deployment is the practice of frequently releasing or deploying code that has been verified through external models such as automated tests. It ensures that software is always in a deployable state and accelerates the release process, reducing bottlenecks and improving feedback speed.

Core Concepts

  • Fully Automated Releases: No manual intervention after code passes CI tests.
  • Reduced Deployment Risk: Smaller, frequent updates lower the chance of major failures.
  • Faster User Feedback: Changes reach users quickly, allowing faster improvements.
  • Requires Strong Testing Culture: Automation must be reliable to avoid pushing broken code.
  • Cost Heavy: the process of deployment can incur significant expenditure depending on workflow models and configuration.

The Goal

To automate deployment and deliver practical code and/or products.

Common CD Workflow

# Developer pushes changes to main branch
git push origin main

# CI pipeline runs:
# 1. Install dependencies
# 2. Build production-ready artifacts (e.g., Docker image)
# 3. Run integration and end-to-end tests
# 4. Report success or failure

# CD pipeline runs automatically if CI passes:
# 1. Deploy artifacts to production environment
# 2. Run post-deployment smoke tests
# 3. Monitor logs and metrics for issues

Examples of CD Platforms

Vercel is a web hosting platform that supports frontend development and continuous deployment.

Netlify is a cloud platform for automating modern web projects, with integrated continuous deployment from Git.

Railway is a deployment platform for hosting apps, databases, and services with automatic Git-based deployment.

Render is a unified cloud platform that offers hosting, databases, and continuous deployment from Git repositories.
