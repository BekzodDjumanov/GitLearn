Tags

In Git, tags are labels that reference a specific commit, often used to mark important points in history. They convey essential information and play a key role in branch names. It's important to note that they cannot be changed once created.

  • v1.0
  • v5.0
  • v7.5

Common Commands

Creating a tag

$ git tag v1.0

Creating an annotated tag

$ git tag -a v1.0 -m "Release version 1.0"

You must push tags when created

$ git push origin v1.0

List tags

$ git tag

Delete tags

$ git tag -d v1.0
