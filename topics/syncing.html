Syncing

Syncing repositories refers to the process of maintaining changes between a local and remote repository. This is done by pulling new changes from the remote and pushing your local changes back to it.

Recall

Common Commands

Sends commits from local to remote branch

$ git push <remote> branch>

Fetches changes from remote and merges them into local branch

$ git pull <remote> branch>

Download changes from remote without merging

$ git fetch <remote>

Best Practices

  • Pull before you push: Prevents conflicts by making sure you’re working on the latest version.
  • Check status often: Run git status to see if your local branch is ahead, behind, or has conflicts.
  • Commit before syncing: Git won’t push uncommitted changes.
  • Resolve conflicts quickly: If pull results in merge conflicts, fix them before pushing.

Basic Workflow

# Pull to get any remote changes
$ git pull origin main

# Make changes locally and commit them
$ git add .
$ git commit -m "Update feature"

# Push changes back to remote
$ git push origin main

Think of the workflow as kicking in futball, you must receive the ball (pull) in order to kick toward the keeper (push).

Summary

Syncing allows developers to maintain consistent code within their local and remote repositories.
