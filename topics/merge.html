Merging

Merging allows two branches to become unified, integrating changes from one branch into another. It combines the histories of two branches, allowing you to bring feature or fix branches back into your main branch.

  • Fast-forward merge: If the current branch is directly behind the target branch, Git simply moves the pointer forward — no new commit is created.
  • Three-way merge: When branches have diverged, Git creates a new merge commit that combines the changes from both branches.
  • Merge conflicts: Occur if the same parts of a file were changed differently in both branches. You’ll need to manually resolve these conflicts.

Common Commands

Merge a branch into your current branch

$ git merge feature-branch

Abort a conflict merge

$ git merge --abort

Shows a visual graph of branches and merges in your commit history

$ git log --graph --oneline

Summary

"git merge" allows a user to combine two branches with their history and changes into one, maintaining a clean and collaborative workflow.
