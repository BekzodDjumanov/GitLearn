Merge Conflicts

A merge conflict happens when Git can’t automatically combine changes from two branches because the same part of a file was changed differently in each branch. These discrepancies can be resolved in a variety of ways by aborting or reverting commits.

Why do Merge Conflicts Occur?

  • Two branches modify the same line of a file differently.
  • Changes overlap or contradict each other.
  • Git doesn’t know which version is correct.

Common Commands

Aborting a merge

$ git merge --abort

Using mergetool for resolution

$ git mergetool

Reverting a commit

$ git revert <commit>

Summary

A merge conflict occurs when Git cannot automatically reconcile differences because two branches have conflicting changes to the same part of a file.
