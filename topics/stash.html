Git Stash

Stash is a component in Git that stores changes that have not been committed, allowing users to assess their software without losing their work. It’s like a “clipboard” for uncommitted changes — you can stash them away and bring them back later.

  • Saves tracked file changes (and optionally untracked files).
  • Clears the working directory to match the last commit.
  • Useful when you need to switch branches or pull changes without committing your in-progress work.

Common Commands

Stores tracked changes

$ git stash

Stores tracked and untracked changes

$ git stash -u

Shows all stashes

$ git stash list

Apply the most recent stash but keep it in the list

$ git stash apply

Apply the most recent stash and remove it from the list

$ git stash pop

Delete a specific stash

$ git stash drop

Summary

git stash saves unsaved changes for later, allowing you to work on other work without losing progress.
