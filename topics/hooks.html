Hooks

Git Hooks are scripts that Git automatically executes before or after certain events, like committing, merging, or pushing. They help automate tasks and enforce rules in your workflow.

Types of Hooks

Client-side hooks

  • pre-commit: Runs before a commit is created.
  • prepare-commit-msg: Before commit message editor
  • commit-msg: Validates or modifies commit messages.
  • post-commit: After commit is created
  • pre-push: Runs before push is commited.

Server-side hooks

  • pre-receive: Checks incoming commits before accepting a push.
  • post-receive: Runs after a successful push, often used for deployments.
  • update: Runs for each branch or ref updated
  • post-update: Runs after each branch of ref is updated

Summary

"git hooks" are scripts that run within specified times in a git workflow
